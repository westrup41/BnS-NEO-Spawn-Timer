# Выпуск новой версии

1. Измените `APP_VERSION` в `config.py` и загрузите исходники в ветку `main`.
2. Создайте тег с тем же номером, например `v4.1`, и отправьте его на GitHub.
3. Workflow `Build and release Windows app` сам соберёт exe, создаст SHA-256,
   GitHub attestation и приложит файлы к Release.

Автообновление принимает exe только при совпадении SHA-256 из файла
`BnS-NEO-Spawn-Timer.exe.sha256` (либо digest GitHub API).

## Подпись Windows

Для доверенной подписи нужен личный Authenticode-сертификат. После его получения
добавьте в GitHub Actions secrets:

- `WINDOWS_CERTIFICATE_BASE64` — PFX в Base64;
- `WINDOWS_CERTIFICATE_PASSWORD` — пароль PFX.

Без этих secrets сборка остаётся рабочей, но Windows не покажет подтверждённого
издателя. Закрытый сертификат нельзя хранить в репозитории.

`BnS-NEO-Spawn-Timer-Admin.exe` предназначен только для владельца и никогда не
загружается в публичный Release. Обычный workflow собирает публичный exe без
`admin_build.flag`, поэтому кнопка администрирования в нём отсутствует.

## Администрирование чата

Кнопка `Админ` показывается только на компьютере владельца, где в локальной папке
программы находится `admin_private_key.pem`. Этот файл и сертификат подписи нельзя
публиковать. Команды очистки, удаления и глобальной блокировки подписываются ключом;
клиенты отвергают поддельные команды.
